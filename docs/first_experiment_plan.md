# 첫 실험 계획

## 목표

논문 PDF 1편에서 물성 정보를 구조화해서 JSON으로 뽑는다.

## 입력

- 공개 소재 논문 PDF 1개

## 출력

아래 필드를 가진 구조화 레코드 배열:

- `material_name`
- `composition`
- `property_name`
- `property_value`
- `property_unit`
- `test_condition`
- `source_page`
- `source_excerpt`

## 성공 기준

- 주요 물성 3개 이상 추출
- 단위가 같이 붙음
- 최소 1개 이상의 실험 조건이 같이 붙음
- 사람이 읽었을 때 "검수 가능한 수준"임

## 실패 기준

- 물성명/값 매칭이 자주 어긋남
- 단위 누락이 많음
- 조성과 물성이 섞여서 나옴

## 순서

1. PDF 텍스트 추출
2. 표/문단 단위로 분리
3. LLM에 스키마 기반 추출 요청
4. JSON 저장
5. 수동 검수

## 메모

- 첫 실험은 너무 어려운 제트엔진/반도체 특허보다, 표준 물성표가 있는 금속/합금 논문이 좋다
- 목적은 사업화가 아니라 기술 가능성 확인이다
